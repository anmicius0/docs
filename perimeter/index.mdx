<Note>
  **核心目標：** 邊界主動防禦的理念是「拒敵於國門之外」——在惡意組件進入開發流程之前，透過 AI 驅動的自動偵測與隔離機制即時攔截，每月可識別並封鎖逾 2,100 個惡意開源組件，從源頭防止惡意程式碼污染供應鏈。Nexus Repository 作為內部可信儲存庫，與 Sonatype Firewall 協同運作，確保所有二進位檔案、容器映像與建置產物均來自經過驗證的單一可信來源。
</Note>

## A.1 — Sonatype Firewall 策略執行與攔截

- **整合產品**：Nexus Repository（儲存）、Sonatype Firewall（過濾）
- **測試環境**：Python (pip), Javascript (npm)

**驗證說明：** 惡意開源套件難以透過肉眼辨識，若流竄至開發或部署階段，清除成本極高。Sonatype Firewall 在套件請求進入內部 Nexus 儲存庫的「第一道關卡」自動執行策略審查。一旦確認為高風險或惡意組件，即刻阻斷下載，確保開發者環境的安全。

## A.3 — 命名空間保護與依賴混淆防護 (Dependency Confusion)

- **整合產品**：Nexus Repository

**驗證說明：** 「依賴混淆」是近年常見的供應鏈攻擊手法。當駭客在公共套件庫上傳與企業內部私有套件（如 `com.company.auth`）同名但版本號更高的惡意套件時，自動化建置系統極易遭到欺騙而下載惡意版本。Nexus Repository 透過**路由規則 (Routing Rules)** 徹底封堵此漏洞：管理者可透過正規表示式，強制規定特定命名空間僅能自內部託管倉庫讀取，嚴禁向外部代理請求，從架構面切斷攻擊路徑。
